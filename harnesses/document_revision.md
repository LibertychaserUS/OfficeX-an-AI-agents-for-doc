# Document Revision Harness

Use this harness when review feedback must be converted into an auditable patch
flow against a copied document target.

## Inputs

- verified review feedback
- issue ledger or equivalent executable work items
- copied target document

## Outputs

- live anchor snapshot
- patch specification
- execution report
- revision audit report

## Guardrails

- do not jump straight from comments to edits
- keep patch scope explicit
- verify before marking an issue fixed
